What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It is an important concept in cloud computing that helps developers automate the process of building, testing, and delivering applications. With Continuous Integration, any new code pushed to the repository is automatically checked to make sure it works correctly. Continuous Deployment then takes the approved changes and publishes them to the live environment. This approach saves time, reduces mistakes, and keeps software always up to date.

My Pipeline

When I push my project to the main branch on GitHub, a workflow is automatically triggered using GitHub Actions. The workflow checks that all required files exist and prepares the project for deployment. After the checks are completed successfully, the website is published directly to GitHub Pages. This means any update I make becomes live automatically without manual uploading.
