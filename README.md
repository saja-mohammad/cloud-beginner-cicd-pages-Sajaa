## 📌 Project Overview

This project demonstrates how to build and deploy a simple static website using GitHub Pages integrated with a CI/CD pipeline powered by GitHub Actions.

The goal of this assignment is to understand how automation works in modern cloud environments. Instead of manually uploading files every time a change is made, the deployment process is fully automated through a workflow configuration.

---

## What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment.

Continuous Integration (CI) ensures that every change pushed to the repository is automatically checked and validated. This helps detect errors early and keeps the project stable.

Continuous Deployment (CD) takes the validated changes and automatically publishes them to a live environment. In this project, GitHub Pages acts as the hosting environment, and deployment happens without manual intervention.

This approach improves efficiency, reduces human errors, and keeps the application always synchronized with the latest updates.

---

## ⚙️ How My Pipeline Works

1. I push changes to the `main` branch.
2. GitHub Actions automatically triggers the workflow defined in `.github/workflows/deploy.yml`.
3. The workflow verifies that the required project files exist.
4. If everything runs successfully, the site is deployed to GitHub Pages.
5. The live website updates automatically with the new changes.

This process reflects a simplified real-world CI/CD pipeline used in cloud-based systems.
